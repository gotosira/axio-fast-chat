Error page 
Error page is the page which informs users about technical problems. This might be shown as standard error code or HTTP Status code to indicate server responses. There are two types of error code: Error 4xx and Error 5xx 
#Error 4xx
Error 4xx is to indicate error happened from users or the error can be solved by users. 
##Contexts
* When users send a malformed request, the server can’t understand the request, such as invalid format or missing parameter. (Error 400 - Bad request) 
* When users are unauthenticated or provide incorrect credentials. (Error 401 - Unauthorized)
* When users request something, servers deny it because of access rights limitations. (Error 403 - Forbidden) 
* When users input the wrong URL for a website. (Error 404 - Not found)
* When users request something that server can’t return a response because of criteria unmatch. (Error 406 - Not acceptable)
* When users take too long to send the request in the specific time session. (Error 408 - Request timeout)
* When users send too many requests in a given time period until server overload. (Error 429 - Too many requests)
##Principles
* Inform users what happens and how to do with clear and concise
* Communicate in neutral, not blame or mention it as mistakes

#Error 5xx 
Error 5xx is to indicate an error occurred from server or technical issues that users can’t solve by themselves.
##Contexts
* When the server can’t process the result or fulfill the request because of unexpected conditions. (Error 500 - Internal server error)
* When a server receives an invalid response from an upstream server. (Error 502 - Bad gateway)
* When a server is temporarily unable to handle the request resulting from different causes, such as overload working in server or server is under maintenance. (Error 503 - Server unavailable)
* When the server doesn’t receive a timely response from the upstream server. (Error 504 - Gateway timeout)
##Principles
* Inform users what happens and how to do with clear and concise
* Communicate in neutral.
* For technical or specialist users, jargon terms with error code is acceptable, such as Error 400 Bad request, or Error 500 Internal server error.
